Judging system

Results

  • Accepted: Your answer is accepted
  • Runtime Error: Your program broke when executing (illegal pointer, stack overflow, unhandled exception or non-zero main function return value, etc.).
  • Time Limit Exceeded: Your program spent too much time.
  • Memory Limit Exceeded: Your program used too much memory.
  • Compile Error: Your program couldn't be compiled.
  • Wrong Answer: Your answer is wrong.
  • System Error: The judging system is broken, please wait for re-judging.
  • Waiting: Your answer is waiting for judging.

Compiling args

  • C（GCC 4.8）
    • gcc -DONLINE_JUDGE -O2 -w -std=c99 {src_path} -lm -o {exe_path}main
  • C++（G++ 4.3）
    • g++ -DONLINE_JUDGE -O2 -w -std=c++11 {src_path} -lm -o {exe_path}main
  • Java（Oracle JDK 1.7）
    • //compile
javac {src_path} -d {exe_path}
//run
java -cp {exe_path} Main

Frequent asked

  • Use standard IO if not specified.
  • Use int main() for C/C++ code, and the program must return 0;.
  • Use long long for 64-bit integers in C/C++, and use cin/cout or %lld to print them. Using __int64 may cause Compile Error.
  • Running time stands for the total CPU time, and used memory is the maximum memory used.
  • The judging server compares your answer with the standard answer after removing the TRAILING SPACES. If they are not the same, you'll get a Wrong Answer. The spaces after lines and empty lines are NOT ignored. There aren't any Format Errors.
